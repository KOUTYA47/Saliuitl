## D-2026-02-02: 復元画像保存機能をsaliuitl.py本体に追加

### 背景（Background）
- ユーザーから復元画像の出力を要求された
- 当初、別スクリプト（save_recovered_images.py）を作成したが、復元ロジックが複雑で正確に再現できなかった

### 検討した選択肢（Options）
- Option A: 別スクリプトで復元ロジックを再実装
- Option B: saliuitl.py に画像保存オプションを追加
- Option C: saliuitl.py の出力をパイプして後処理

### 判断（Decision）
- 採用した選択肢：Option B（saliuitl.py本体に追加）

### 判断理由（Rationale）
- 復元処理が反復的で複雑（β値ごとにDBSCANクラスタリング→インペインティング→再検出）
- 本体に追加すれば、`in_img`（復元画像）に直接アクセス可能
- 追加オプション（--save_images）でデフォルト動作に影響なし

### 却下理由（Why not others）
- Option A: 別スクリプトでの再実装は不正確になった（マスクが大きすぎる問題）
- Option C: 復元途中の画像はstdoutに出力されないため不可

### 影響範囲（Impact）
- saliuitl.py に3つの引数追加（--save_images, --save_images_dir, --save_images_limit）
- matplotlib の条件付きインポート追加
- 復元処理後に画像保存コード追加（約50行）

### 備考（Notes）
- 物体検出（INRIA/VOC）のみ対応、分類タスク（CIFAR/ImageNet）は未対応
- Detection Maskの可視化に問題あり（特徴マップサイズが小さい）

## D-2026-02-05: Oracle TestによるボトルネックをマスクRR精度と特定

### 背景（Background）
- R-2026-02-02-Eで「成功」判定でもパッチが視覚的に残存する問題を発見
- 原因候補: (A) マスク精度の問題 vs (B) インペインティング手法の問題
- 定量的な切り分けが必要

### 検討した選択肢（Options）
- Option A: マスク生成アルゴリズムの詳細分析
- Option B: Oracle Testで上限性能を測定し、差分からボトルネックを特定
- Option C: 複数のインペインティング手法（zero, mean, diffusion）を比較

### 判断（Decision）
- 採用した選択肢：Option B（Oracle Test）

### 判断理由（Rationale）
- Oracle Testは「マスクが完璧な場合」の上限性能を示す
- Biharmonicとの差 = マスク不正確性による損失を直接測定可能
- 全データセットで実施することで、タスク間の差異も明確化

### 却下理由（Why not others）
- Option A: 実装詳細の分析は時間がかかり、定量的評価が困難
- Option C: インペインティング手法の比較はマスク精度の問題と混在する

### 影響範囲（Impact）
- 物体検出: インペインティング品質の影響が大きい（Oracle vs Bio: **+22.8%**）
- 画像分類: インペインティング品質の影響は限定的（**+0.8%**）
- **結論**: 物体検出ではインペインティング品質改善に余地あり、ただしマスク精度の問題は別途改善が必要

### 備考（Notes）
- 高解像度特徴マップ（Layer 10: 52×52）の使用が有望
- ただしADモデルの再学習が必要になる可能性あり

## D-2026-02-05-B: 改善優先順位の決定

### 背景（Background）
- Oracle Test結果の詳細考察を完了
- 物体検出と画像分類で改善効果が大きく異なることが判明
- リソース配分の優先順位を決定する必要がある

### 検討した選択肢（Options）
- Option A: インペインティング手法の改善を優先（diffusion等）
- Option B: マスク精度の改善を優先（高解像度特徴マップ）
- Option C: 検出率の改善を優先（DBSCANパラメータ、AD再学習）
- Option D: タスク特性に応じた使い分け

### 判断（Decision）
- 採用した選択肢：Option D（タスク特性に応じた使い分け）

### 判断理由（Rationale）
1. **物体検出**: 検出率 > マスク精度 > インペインティング品質の順で改善効果が大きい
2. **画像分類**: マスク精度改善の効果は限定的（+0.8%）、別アプローチが必要
3. Oracle Testにより、ボトルネックの階層構造が定量的に明らかになった

### 却下理由（Why not others）
- Option A: インペインティング品質はLevel 3であり、上位のボトルネックを解消しないと効果が薄い
- Option B: 画像分類では効果が薄い（+0.8%）
- Option C: 画像分類には適用しにくい

### 影響範囲（Impact）
- 今後の研究開発リソース配分
- 論文の議論セクション
- 発表スライドの改善提案部分

### 備考（Notes）
- 詳細考察: `docs/notes/oracle_test_analysis.md`
- 物体検出の検出率向上が最優先課題として明確化

## D-2026-02-05-D: AD汎化性能の検証が必要

### 背景（Background）
- ADはデータセット固有・攻撃パターン固有で学習されている
- 未知のデータセット・攻撃パターンへの汎化性能は未検証
- INRIA trigの検出率54.5%が低い（学習データに三角形パッチが含まれない可能性）

### 判断（Decision）
- クロスデータセット・クロス攻撃評価でADの汎化性能を検証する

### 判断理由（Rationale）
1. 実運用では未知の攻撃に遭遇する可能性が高い
2. Saliuitlの適用範囲・限界を明確化する必要がある
3. 関連研究（NutNet, DIFFender等）では汎化性能が重視されている

### 影響範囲（Impact）
- 論文の「汎用性」主張に対する検証
- Saliuitlの位置づけの明確化
- 改善方向性の検討

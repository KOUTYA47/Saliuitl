### Saliuitl: Ensemble Salience-Guided Recovery of Adversarial Patches against CNNs

- **著者**: Victorica et al. (KTH Royal Institute of Technology)
- **会議**: CVPR 2025
- **概要**: 敵対的パッチ攻撃に対する検出・復元手法

---

## Slide 3: 背景・課題

### 敵対的パッチ攻撃

- 画像にパッチを貼り付けてCNNを誤認識させる攻撃
- 物理世界で実現可能（自動運転、顔認証への脅威）

### 既存防御手法の問題

| 問題 | 説明 |
|------|------|
| Blind Recovery | 全画像に復元処理 → コスト増大、クリーン画像劣化 |
| 固定閾値 | 単一閾値では多様な攻撃に対応不可 |
| Certifiable Defense | 高い保証だが計算コストが非常に高い |

---

## Slide 4: 提案手法 Saliuitl

### 2段階パイプライン

```
┌─────────────────────────────────────────┐
│ 1. 検出ステージ                           │
│    入力 → 特徴マップ → 複数閾値で二値化    │
│    → DBSCAN属性抽出 → Attack Detector    │
│    → 攻撃あり / なし                      │
└─────────────────────────────────────────┘
              ↓ 攻撃あり
┌─────────────────────────────────────────┐
│ 2. 復元ステージ                           │
│    マスク生成 → Biharmonicインペインティング │
│    → 再予測 → 検出回復                    │
└─────────────────────────────────────────┘
```

### 特徴

- **アンサンブル閾値**: 複数閾値で堅牢性向上
- **攻撃時のみ復元**: クリーン画像の精度維持

## Slide 10: 衝撃の発見

### 出力して分かったこと

**「成功」判定でもパッチが視覚的に残っている！**

| 画像 | 判定 | 検出数変化 | パッチ |
|------|------|-----------|--------|
| 000001 | Failed | 2→3→6 | ❌ 残存 |
| 000006 | **Success** | 3→3→8 | ❌ 残存 |

### 重要な気づき

| Recovery Rate | 視覚的状態 |
|---------------|-----------|
| Success = 検出回復 | ≠ パッチ除去 |

**評価指標と期待する結果にズレがある**

→ なぜこうなるのか？**原因分析**へ

## Slide 11: 原因分析（1）設計上の理由

### 可視化マスク vs 実際のインペインティング

```
┌─────────────────────────────────────────────────────┐
│ 可視化（Mask Overlay）:                             │
│   全βの結果を蓄積 → パッチ全体をカバー             │
├─────────────────────────────────────────────────────┤
│ 実際のインペインティング:                           │
│   各β単独で実行 → 毎回オリジナル画像に戻る         │
│   → 部分的なマスクのみ適用                         │
└─────────────────────────────────────────────────────┘
```

### これはバグではなく意図的な設計

| 論文の目標 | 検出の回復（Recovery Rate） |
|-----------|---------------------------|
| 設計思想 | 複数βを試し、1つでも成功すればOK |
| 検出ボックス | 全βから蓄積してNMS |
| 画像 | 累積的に復元しない |

→ **評価指標上は合理的だが、視覚的除去は目標外**

**図**: `slides_material/inpainting_design.png`

## Slide 12: 原因分析（2）技術的制約

### 制約1: マスクサイズ上限（5.5%）

```
マスク上限 = 画像サイズ × 0.5 / 9 ≈ 5.5%
         = 約 97×97 ピクセル
```

| パッチサイズ | 回復 |
|-------------|------|
| < 5.5% | 試行可能 |
| > 5.5% | **スキップ → 回復不可** |

→ **大きなパッチには対応できないアーキテクチャ的制約**

### 制約2: 解像度ミスマッチ

```
特徴マップ: 26×26 → 入力画像: 416×416（×16）
```

### 制約3: Biharmonicの特性

- 曲率最小化 → ぼやける
- テクスチャ再現不可

**図**: `slides_material/mask_size_limit.png`, `slides_material/resolution_mismatch.png`

## Slide 14: 結論

### 再現結果

| 指標 | 結果 |
|------|------|
| Recovery Rate | 概ね再現（VOCで±2%） |
| nmAP | 再現可能（値は高め） |
| 計算時間 | 論文の23-30% |

### 発見した限界

1. **視覚的パッチ除去は達成されていない**
   - 可視化マスクと実際のインペインティングにギャップ
2. **大きなパッチ（>5.5%）には対応不可**
   - マスクサイズ上限によるアーキテクチャ的制約
3. 解像度ミスマッチによるマスク精度の限界

### 今後の展望

- 高解像度特徴マップの活用
- セグメンテーションベースのマスク生成
- 現代的インペインティング手法（LaMa等）

# Session Notes - 2026-02-05

## 実施内容

### Oracle Inpainting Test（全データセット）

- **目的**: 全15パターンでOracle Testを実施し、biharmonic再現実験との差分を定量化
- **実施パターン**: 物体検出7シナリオ + 画像分類8シナリオ（INRIA moはデータなし）

### 実験結果

#### 物体検出（INRIA, VOC）
- 平均Oracle RR: 72.6%
- 平均Biharmonic RR: 49.8%
- **平均改善: +22.8%**
- 最大改善: VOC mo (+44.4%)

#### 画像分類（CIFAR, ImageNet）
- 平均Oracle RR: 75.0%
- 平均Biharmonic RR: 74.1%
- **平均改善: +0.8%**
- 一部でOracle < Biharmonic（統計的変動）

### 主要な発見

1. **物体検出ではOracleが効果的**: インペインティング品質改善の余地あり
2. **画像分類ではOracle効果が限定的**: Biharmonicで十分な性能
3. **検出率がRRの上限を決定**: INRIA trig Detected=54.5%でOracle RR=54.5%に制限
4. **Oracle Inpaintingはマスク精度を改善しない**: インペインティング品質のみ向上

### 技術的問題と解決

- **Docker network exhaustion**: `docker compose run`で"address pools fully subnetted"エラー
  - 解決: `docker run --network host`を使用
- **effective_files path duplication**: パスが二重連結されるバグ
  - 解決: ファイル名のみを指定（`effective_1p.npy`）

## 1. 実験の位置付け

Oracle Inpainting Test（`--inpaint oracle`）は、システムが生成したマスク領域にクリーン画像のピクセルをそのまま貼り付ける手法である。

**重要な注意**: Oracleは**インペインティング品質の上限**を示すが、**マスクの位置・形状は変わらない**。

## 3. 考察

### 3.1 Oracle Inpaintingの意味を再確認

Oracle Inpaintingは「マスク領域にクリーン画像をコピー」する手法である。

**改善される要素**:
- インペインティングの品質（ぼやけ、アーティファクトなし）

**改善されない要素**:
- マスクの位置精度
- マスクの形状精度
- 検出率（Detected）

### 3.2 物体検出でOracleが効果的な理由

物体検出では平均+22.8%の改善が見られた。これは：

1. **Biharmonicのぼやけ問題**: 物体検出はエッジや形状に敏感であり、ぼやけが検出精度に影響
2. **マスク領域外への影響**: Biharmonicは周囲のピクセルから補間するため、マスク外にも影響が波及
3. **パッチサイズの相対的小ささ**: 物体検出ではパッチが小さいため、Biharmonicの品質低下が顕著

### 3.3 画像分類でOracleが効果的でない理由

画像分類では平均+0.8%の改善に留まり、一部でマイナスとなった。原因：

1. **分類は位置に鈍感**: CNNの分類は特徴の有無を見るため、インペインティング品質の影響が小さい
2. **Biharmonicで既に十分**: 画像分類ではBiharmonicの品質で十分な復元が達成されている
3. **サンプル数の統計的変動**: 一部のマイナスは小サンプルによる変動の可能性

### 3.4 Oracle RR < Biharmonic RRのケース

CIFAR 4p, CIFAR trig, ImageNet 1pでOracleがBiharmonicより低下。

**可能な説明**:
1. **検出率の違い**: 異なる実行間での検出率のばらつき
2. **統計的変動**: サンプル数が限られているため
3. **実験条件の微細な違い**: Dockerコンテナの状態など

### 3.5 検出率（Detected）の重要性

INRIA trig: Detected = 54.5%と低く、Oracle RRも54.5%に制限された。

**示唆**:
- 検出されないサンプルは復元対象にならない
- **検出率の向上がRR向上の前提条件**
- マスク精度やインペインティング品質以前の問題

## 4. Saliuitl手法の限界（修正版）

### 4.1 構造的限界

1. **マスク生成の精度**: 26×26 → 416×416のアップスケーリングによる位置誤差
2. **検出率の制約**: 検出されないサンプルは復元不可
3. **DBSCANの等方性**: 非矩形パッチ（三角形）への対応が不十分

### 4.2 達成可能な上限（修正）

Oracle Test結果から：
- **物体検出**: 平均72.6%（最大100%、最小45.0%）
- **画像分類**: 平均75.0%（最大92.9%、最小40.0%）

**注意**: これは「インペインティング品質の上限」であり、「マスク精度の上限」ではない。

---

## 5. 改善の方向性（優先度順）

### 5.1 検出率の向上（最優先）

INRIA trigでDetected=54.5%と低いことから、検出アルゴリズムの改善が必要：
- Attack Detectorの閾値調整
- ADモデルの再学習
- 三角形パッチ用の訓練データ追加

### 5.2 マスク精度の向上

物体検出で+22.8%の改善余地があることから：
- 高解像度特徴マップ（Layer 10: 52×52）の使用
- 膨張パラメータの最適化
- 閾値選択アルゴリズムの改善

### 5.3 インペインティング品質の向上（低優先度）

画像分類では効果が限定的であることから、優先度は低い。
ただし、視覚的品質を重視する場合は：
- LaMa、DeepFillなどのGAN-basedインペインティング
- Diffusion-basedインペインティング

---

## 6. 結論

1. **物体検出ではOracleが効果的**（平均+22.8%）→ インペインティング品質の改善余地あり

2. **画像分類ではOracle効果が限定的**（平均+0.8%）→ Biharmonicで十分

3. **検出率がRRの上限を決定**: 検出されないサンプルは復元不可

4. **マスク精度の問題は別途調査が必要**: Oracleではマスク精度は評価できない

5. **改善優先度**: 検出率向上 > マスク精度向上 > インペインティング品質向上

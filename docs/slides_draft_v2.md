# 読んだ論文

## Saliuitl: Ensemble Salience-Guided Recovery of Adversarial Patches against CNNs

- **著者**: Victorica et al. (KTH Royal Institute of Technology)
- **会議**: CVPR 2025
- **概要**: 敵対的パッチ攻撃に対する**検出・復元**手法

---

## 敵対的パッチ攻撃とは

| 攻撃の仕組み | 影響 |
|-------------|------|
| 画像にパッチを貼り付ける | CNNが誤認識 |
| 物理世界で実現可能 | 自動運転・顔認証への脅威 |

### 攻撃パッチの種類

| 種類 | 説明 |
|------|------|
| 1p / 2p / 4p | 正方形パッチ × n個 |
| trig | 三角形パッチ |
| mo | 各物体に1パッチ（multi-object）|

**図**: 攻撃前後の検出結果比較

# Slide 3: Saliuitlの手法

---

# Saliuitlの手法（概要）

## 2段階パイプライン

```
[入力画像]
    ↓
[特徴マップ抽出] → [複数閾値で二値化] → [DBSCAN属性抽出]
    ↓
[Attack Detector] → 攻撃あり？
    │
    ├─ NO  → そのまま出力
    │
    └─ YES → [マスク生成] → [インペインティング] → [復元出力]
```

## キーポイント

- **アンサンブル閾値**: 複数の閾値βで堅牢性向上
- **攻撃時のみ復元**: クリーン画像の精度を維持
- **Biharmonicインペインティング**: マスク領域を滑らかに補間

# 実験設定

## データセット・モデル

| タスク | データセット | モデル | 画像数 |
|--------|------------|--------|--------|
| 物体検出 | INRIA | YOLOv2 | 30枚 |
| 物体検出 | VOC | YOLOv2 | 29枚 |
| 画像分類 | CIFAR-10 | ResNet-50 | 30枚 |
| 画像分類 | ImageNet | ResNet-50 | 30枚 |

※ 論文のフルデータセットのサブセットで評価

## 評価指標

| 指標 | 定義 |
|------|------|
| **RR** (Recovery Rate) | 復元に成功した攻撃画像の割合 |
| **LPR** (Lost Prediction Rate) | クリーン画像で誤検出が発生した割合 |
| **nmAP** | 復元後mAP / ベースラインmAP |

## パラメータ

- 閾値集合サイズ |B| = 20、インペインティング = biharmonic

# Slide 8: 成功の定義と乖離

---

# 「成功」の定義 vs 視覚的期待

## 論文での「成功」の定義

```
復元後の検出結果がクリーン検出と IoU ≥ 0.5 でマッチ
```

→ **検出が回復すれば「成功」**（パッチ除去は問わない）

## 視覚的な期待

```
パッチが除去され、元の画像に近づく
```

## この乖離の意味

| 観点 | 説明 |
|------|------|
| 検出回復 | パッチが残っていても達成可能 |
| 視覚的除去 | 論文の評価対象外 |

→ **評価指標（RR）と視覚的品質は別物**

## 原因候補

- **(A) マスク精度の問題**: マスクがパッチを捕捉できていない
- **(B) インペインティング品質**: 復元がぼやける

→ **どちらがボトルネックか？ → Oracle Testで検証**

# Oracle Testによるボトルネック特定

## Oracle Inpaintingの定義

```
マスク領域にクリーン画像のピクセルをそのまま貼り付ける
```

→ **「インペインティング品質の理論上の上限」を示す**

## 実験: 全15パターンで比較

Biharmonic（通常手法）vs Oracle（理論上限）

## 結果

| タスク | Biharmonic RR | Oracle RR | 改善幅 |
|--------|---------------|-----------|--------|
| **物体検出** | 49.8% | 72.6% | **+22.8%** |
| **画像分類** | 74.1% | 75.0% | +0.8% |

## 解釈

| タスク | 意味 |
|--------|------|
| 物体検出 | マスク精度改善で**大きな効果**が期待できる |
| 画像分類 | Biharmonicで**既に十分**（改善余地なし）|

# Slide 10: ボトルネックの特定

---

# ボトルネック階層構造の発見

## 階層モデル

```
              Recovery Rate（復元率）
                      ↑ 制約
    ┌─────────────────────────────────────────┐
    │ Level 3: インペインティング品質          │
    │          改善余地: 5-15%                 │
    └─────────────────────────────────────────┘
                      ↑ 制約
    ┌─────────────────────────────────────────┐
    │ Level 2: マスク精度                      │
    │          物体検出: +22.8% ← 改善余地大   │
    │          画像分類: +0.8%  ← 改善余地小   │
    └─────────────────────────────────────────┘
                      ↑ 制約
    ┌─────────────────────────────────────────┐
    │ Level 1: 検出率 ← 最重要ボトルネック     │
    │          trig攻撃: 54-67%まで低下        │
    └─────────────────────────────────────────┘
```

## 具体例

| ケース | 数値 | 意味 |
|--------|------|------|
| INRIA trig | 検出率54.5% | どんなに改善してもRR≤54.5% |
| VOC mo | Oracle効果+44.4% | マスク精度が特に低かった |

## 結論: 改善優先順位

**検出率 > マスク精度 > インペインティング品質**

# Slide 11: Saliuitlの限界

---

# Saliuitlの限界と課題

## 限界1: 検出率の問題

| 攻撃パターン | 検出率 | 問題 |
|-------------|--------|------|
| 1-patch, 2-patch | 86-100% | 良好 |
| **trig (三角形)** | **54-67%** | 低い |

→ 学習データに含まれない形状への汎化が困難

## 限界2: データセット固有のAD

- Attack Detectorは**データセットごとに学習**
- 新しいドメインには**再学習が必要**
- 汎用的な防御ではない

## 限界3: 解像度のミスマッチ

```
特徴マップ: 26×26 → 入力画像: 416×416
スケール比: 16倍 → 1ピクセル誤差 = 16ピクセル誤差
```

→ マスク精度に根本的な限界がある

## 限界4: マスクサイズ上限

- 画像の約5.5%まで（97×97ピクセル）
- 大きなパッチには対応不可

# 結論

## 再現実験の結果

| 項目 | 結果 |
|------|------|
| Recovery Rate | 概ね再現（VOC 1p/2pで±2%）|
| 一部乖離 | trig攻撃で検出率低下により大きく乖離 |

## 独自の発見（本研究の貢献）

### 1. 復元画像の可視化

- **「成功」≠「パッチ除去」** を視覚的に確認
- マスクがパッチを正確に捕捉できていない

### 2. Oracle Testによるボトルネック特定

| タスク | マスク精度改善の効果 |
|--------|---------------------|
| 物体検出 | **+22.8%**（改善余地大）|
| 画像分類 | +0.8%（改善余地小）|

### 3. 改善優先順位の明確化

**検出率 > マスク精度 > インペインティング品質**

## 今後の課題

- 検出率向上（特にtrig攻撃）
- 高解像度特徴マップの活用
- AD汎化性能の検証

# 発表ノート

## Q&A想定

### Q: なぜ画像分類ではOracle効果が小さいのか？

**A**: 画像分類ではパッチが相対的に大きいため、マスクの位置誤差が許容されやすい。また、分類は局所的な特徴に依存しにくく、部分的な復元でも十分な効果がある。

### Q: 検出率を上げるにはどうすればよいか？

**A**: DBSCANパラメータの調整、Attack Detectorの学習データ拡張（三角形パッチを含める）、または検出閾値の調整が考えられる。

### Q: 論文の手法に問題があるということか？

**A**: 問題というより**設計上のトレードオフ**。論文は「検出回復」を目標としており、その点では合理的。視覚的なパッチ除去は論文の評価対象外。

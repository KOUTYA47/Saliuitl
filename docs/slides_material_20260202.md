## 1. プロジェクト概要

### Saliuitlとは

- **目的**: 敵対的パッチ攻撃の検出と復元
- **手法**: 特徴マップのDBSCANクラスタリング + Attack Detector + インペインティング
- **対象**: YOLOv2（物体検出）、ResNet50（画像分類）

### パイプライン

```
入力画像 → 被害者モデル → 特徴マップ抽出 →
β閾値で二値化 → DBSCANクラスタリング →
4属性抽出 → Attack Detector →
検出時: マスク生成 → インペインティング → 復元画像
```

## 3. 問題の発見

### 観察: 「成功」でもパッチが残っている

可視化実験で発見した問題：

| 画像 | 判定 | パッチ除去 | 検出数変化 |
|------|------|-----------|-----------|
| VOC 000001 | Failed | ❌ **残存** | 2→3→6 |
| VOC 000006 | Success | ❌ **残存** | 3→3→8 |
| VOC 000010 | Success | ❌ **残存** | 1→2→4 |

**重要**: 「Success」は検出回復（IoU≥0.5）を意味し、**視覚的なパッチ除去ではない**。

### 問題の根本原因

```
┌────────────────────────────────────────────────────┐
│ 問題の連鎖                                          │
├────────────────────────────────────────────────────┤
│ 1. 特徴マップ解像度が低い (26×26)                    │
│         ↓                                          │
│ 2. マスクが不正確 (16倍アップスケール)                │
│         ↓                                          │
│ 3. パッチの一部しかマスクされない                    │
│         ↓                                          │
│ 4. インペインティングが部分的                        │
│         ↓                                          │
│ 5. パッチのテクスチャが残存                          │
│         ↓                                          │
│ 6. 視覚的には失敗だが検出は回復 → "Success"         │
└────────────────────────────────────────────────────┘
```

## 5. 視覚的回復改善実験

### 実験A: Oracle Inpainting Test

**目的**: マスク精度 vs インペインティング手法の問題切り分け

**手法**: Clean/Attacked画像の差分でGround-truthマスクを生成し、biharmonicを適用

**結果**:

| Dataset | 平均エラー | 結論 |
|---------|-----------|------|
| VOC | 0.63 | **パッチ除去可能** |
| INRIA | 0.23 | **パッチ除去可能** |

**発見**: 正確なマスクがあればbiharmonicでパッチを視覚的に除去できる。ただし結果は**ぼやける**。

### 実験B: Grad-CAM Attention可視化

**目的**: パッチが検出を妨害するメカニズムの理解

**手法**: YOLOv2のLayer 13でGrad-CAMを生成

**結果**:

| 画像 | Clean Score | Attacked Score | Attention変化 |
|------|-------------|----------------|---------------|
| 000001 | 0.884 | 0.670 | 人物→**パッチに移動** |
| 000002 | 0.777 | 0.700 | 電車→**パッチに移動** |

**発見**: 敵対的パッチは検出器のAttentionを**対象物体からパッチ自体にハイジャック**する。

## 6. 核心的発見まとめ

### 発見1: マスク精度が主要ボトルネック

- 現システムのマスクが断片的で不正確
- 26×26→416×416のアップスケール（16倍）が精度を低下
- Oracle maskを使えばパッチ除去は可能

### 発見2: Biharmonicインペインティングの限界

- 曲率最小化アルゴリズム → 結果がぼやける
- テクスチャ・エッジの再構成は不可能
- 大きな領域では品質低下

### 発見3: Attention Hijacking

- 敵対的パッチはモデルのAttentionを奪う
- 対象物体からパッチに注目が移動
- これが検出失敗の根本メカニズム

### 発見4: 「成功」の定義のずれ

- 論文の評価指標（RR, nmAP）は**検出回復**を測定
- **視覚的なパッチ除去**は評価対象外
- 「Success」でもパッチは見える状態で残存

---

## 7. 定義の整理

### 評価指標

| 指標 | 定義 |
|------|------|
| **RR** (Recovery Rate) | 復元成功した effectively attacked 画像数 / effectively attacked 画像総数 |
| **LPR** (Lost Prediction Rate) | Rφ適用後に誤出力となるクリーン画像数 / クリーン画像総数 |
| **nmAP** (normalized mAP) | mAP(復元後) / mAP(クリーンベースライン) |

### 攻撃成功の定義

| タスク | Effective Attack条件 |
|--------|---------------------|
| 物体検出 | クリーン予測の≥1オブジェクトが攻撃後予測の全オブジェクトとIoU < 0.5 |
| 画像分類 | 攻撃後の予測 ≠ クリーン予測 |

## 9. 今後の改善案

### 優先度1: マスク精度改善

| 方法 | 期待効果 |
|------|---------|
| 高解像度特徴マップ (Layer 10: 52×52) | 精度2倍 |
| 膨張パラメータ最適化 (n_dilation=5-7) | カバレッジ向上 |
| 閾値選択改善（平均→最大カバレッジ） | 精度向上 |

### 優先度2: インペインティング改善

| 方法 | 期待効果 |
|------|---------|
| GAN-based (LaMa, DeepFill) | 高品質復元 |
| Diffusion-based | 意味的復元 |

### 優先度3: 攻撃対策

| 方法 | 期待効果 |
|------|---------|
| Adversarial training | 耐性向上 |
| Attention regularization | Hijacking防止 |

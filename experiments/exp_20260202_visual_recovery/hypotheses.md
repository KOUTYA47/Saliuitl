# 視覚的回復品質改善のための仮説

実験ID: `exp_20260202_visual_recovery`
作成日: 2026-02-02

## 調査結果サマリ

### 1. 特徴マップ解像度

| レイヤー | 解像度 | チャンネル数 | 用途 |
|---------|--------|-------------|------|
| Layer 13 (現行) | **26×26** | 256 | AD学習済み |
| Layer 10 | **52×52** | 256 | 高解像度オプション |
| Layer 8 | **52×52** | 256 | 高解像度オプション |

**発見**: 実際の解像度は26×26（13×13ではなかった）。416×416へのアップスケールは16倍。

### 2. インペインティング手法

| 手法 | 実装状態 | 備考 |
|------|---------|------|
| biharmonic | ✅ 完全実装 | scikit-image使用 |
| diffusion | ❌ 未実装 | biharmonicにフォールバック |
| oracle | ❌ スタブのみ | クリーン画像が必要 |
| zero | ✅ 実装 | 0で埋める |
| mean | ✅ 実装 | 平均値で埋める |

### 3. マスク生成パイプライン

```
特徴マップ抽出 → 正規化 → β閾値で二値化 → DBSCAN →
平均化 → nearest補間で拡大 → 膨張（7×7楕円、3回）
```

**調整可能パラメータ**:
- `--n_dilation`: 膨張回数（デフォルト: 3）
- `--dbscan_eps`: クラスタリング半径（デフォルト: 1.0）
- `--dbscan_min_pts`: 最小クラスタサイズ（デフォルト: 4）
- `--inpainting_step`: β閾値数（デフォルト: 5 → 20閾値）

## 仮説リスト

### H1: 高解像度特徴マップによるマスク精度向上

**仮説**: Layer 10（52×52）を使用することで、マスクの位置精度が2倍に向上する。

**根拠**:
- 現在: 26×26 → 416×416（16倍アップスケール）
- 提案: 52×52 → 416×416（8倍アップスケール）
- 1ピクセル誤差の影響が半減

**検証方法**:
1. `saliuitl.py`の`return_feature=13`を`return_feature=10`に変更
2. 同一画像でマスクを比較
3. パッチカバレッジ率を測定

**リスク**: ADモデルの再学習が必要な可能性

---

### H2: 膨張パラメータの最適化

**仮説**: 膨張回数と/またはカーネルサイズの調整でマスクカバレッジを改善できる。

**根拠**:
- 現在: 7×7楕円、3回膨張
- パッチサイズに対して膨張が不十分な可能性

**検証方法**:
1. `--n_dilation`を3→5→7と変化させて比較
2. カーネルサイズを7→11に変更して比較

**評価指標**: パッチカバレッジ率、false positive率

---

### H3: 閾値選択の改善（検出ベース→カバレッジベース）

**仮説**: 現在のβ閾値平均化ではなく、最大カバレッジを与える閾値を選択することでパッチ除去が改善される。

**根拠**:
- 現在: 全β閾値の平均
- パッチは特定の閾値範囲で最も明確に検出される可能性

**検証方法**:
1. 各β閾値でのマスクを個別に可視化
2. 最大クラスタサイズを与える閾値を自動選択
3. 平均 vs 最大選択の比較

---

### H4: Oracle インペインティングで上限性能を確認

**仮説**: Ground-truthマスク（パッチ位置の正解）を使用すれば、視覚的にパッチを除去できる。

**根拠**:
- 現在の問題がマスク精度 vs インペインティング手法のどちらにあるか切り分け可能
- もしoracleでも品質が悪ければ、biharmonic自体の限界

**検証方法**:
1. パッチ位置をGround-truthとして手動/自動で取得
2. oracle マスクでbiharmonicインペインティング
3. 結果を現行マスクと比較

---

### H5: Attention可視化による検出メカニズム理解

**仮説**: モデルのAttention（注目領域）を可視化することで、パッチがどのように検出を妨害しているか理解できる。

**根拠**:
- パッチは検出モデルの注意を逸らすことで攻撃が成功
- Attentionがパッチに集中しているか、対象物体から逸れているかを確認

**検証方法**:
1. Grad-CAM実装
2. Clean/Attacked/Recoveredそれぞれのattention map生成
3. 比較分析

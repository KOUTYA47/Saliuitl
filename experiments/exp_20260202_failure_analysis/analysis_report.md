# 回復画像品質問題の分析レポート

実施日: 2026-02-02
実験ID: `exp_20260202_failure_analysis`

## 概要

回復画像が「ぼやける」「パッチが除去されていない」問題について技術的分析を行った。

## 核心的発見

**パッチは完全に除去されていない** - 全ての「Success」画像でも敵対的パッチがそのまま可視状態で残っている。

| 画像 | 状態 | パッチ除去 | 検出回復 |
|------|------|-----------|---------|
| VOC 000001 | Failed | ❌ 残存 | 2→3→6 |
| VOC 000006 | Success | ❌ 残存 | 3→3→8 |
| VOC 000010 | Success | ❌ 残存 | 1→2→4 |

## 問題の原因分析

### 1. 解像度のミスマッチ（最重要）

**ソース**: `saliuitl.py:853-920`

```
特徴マップ: 13×13 (169ピクセル)
入力画像:  416×416 (173,056ピクセル)
スケール比: 1:1024 (32×32ブロック単位)
```

| 問題 | 影響 |
|------|------|
| 1ピクセル誤差 → 32ピクセル誤差 | マスク境界が不正確 |
| 40×40パッチ ≈ 1.25セル | 精密な位置特定が不可能 |
| 複数回の膨張処理 | マスクが「膨らみすぎ」または「足りない」 |

**コード箇所** (`saliuitl.py:895-910`):
```python
h_factor = self.sized.shape[0] / fms.shape[-2]  # 416/13 = 32
w_factor = self.sized.shape[1] / fms.shape[-1]  # 416/13 = 32
# Each 1x1 feature map cell → 32x32 pixel block in mask
```

### 2. Biharmonicインペインティングの限界

**ソース**: `helper.py:1286-1295`

```python
def biharmonic_inpaint(img, mask):
    inpainted = inpaint_biharmonic(img, mask, channel_axis=-1)
    return inpainted
```

| 特性 | 結果 |
|------|------|
| 曲率最小化アルゴリズム | 滑らか＝ぼやける |
| 小さな傷向け設計 | 大きなパッチには不適 |
| 意味的理解なし | テクスチャ・エッジ再構成不可 |

### 3. マスク生成パイプラインの問題

**ソース**: `saliuitl.py:853-920`

```
特徴マップ → 二値化 → 膨張(4回) → アップスケール → 膨張(16回)
```

可視化画像から確認:
- Inpainting Maskが**断片的**で連続していない
- パッチ全体をカバーできていない
- 周囲の正常領域も誤ってマスクされている（5.8%〜10.3%）

### 4. 最適化目標のずれ

**現在の最適化**: 検出回復（IoU ≥ 0.5）
**ユーザー期待**: 視覚的なパッチ除去

システムは「検出が戻れば成功」と判定するが、パッチ自体は残る。

## 技術的根本原因

```
┌─────────────────────────────────────────────────────────────┐
│                     問題の連鎖                               │
├─────────────────────────────────────────────────────────────┤
│  特徴マップ解像度が低い (13×13)                              │
│         ↓                                                   │
│  マスクが不正確（32×32ブロック単位）                          │
│         ↓                                                   │
│  パッチの一部しかマスクされない                               │
│         ↓                                                   │
│  Biharmonicが部分的にしか処理できない                         │
│         ↓                                                   │
│  パッチのテクスチャが残存                                     │
│         ↓                                                   │
│  視覚的には「除去失敗」だが検出は回復 → "Success"            │
└─────────────────────────────────────────────────────────────┘
```

## 結論

1. **「成功」の定義が視覚的品質と乖離**している
2. **パッチ除去は行われていない** - 検出回復のみ達成
3. **13×13→416×416のスケーリング**が根本的なボトルネック
4. **Biharmonicインペインティング**は大きなパッチに不適

この問題は論文の手法自体の限界であり、コードのバグではない。
